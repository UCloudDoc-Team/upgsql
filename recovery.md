# 恢复
云数据库 UDB PostgreSQL 支持将实例数据恢复至指定时间点或备份版本，以应对误操作、数据丢失或业务回滚等场景。用户可在控制台选择可用的备份文件或指定时间点，通过创建新实例的方式完成数据恢复，恢复过程中不会影响当前业务运行。该功能适用于快速找回历史数据、验证备份有效性及构建测试环境等场景。

## 备份恢复至新实例
**前置条件**

已生成备份文件

**操作步骤** 

1. 访问 [UDB PostgreSQL 实例列表](https://console.ucloud.cn/postgresql/postgresql)，找到目标实例，点击“操作”列中的“详情”按钮。
2. 点击“备份管理”tab页，选择目标备份文件，点击操作列的“恢复至新实例”按钮。

![](/images/1764042175182-8391e220-81fc-4418-b3ed-43f69e892b98.png)



## 回档至新实例
云数据库 UDB PostgreSQL 支持数据库回档。当因误操作导致数据删除或丢失时，可通过“回档至新实例”将数据恢复到指定时间点。系统会基于全量备份及其期间所产生的 WAL 日志进行恢复。

**前置条件**

已成功创建实例，当前处于运行中状态，且数据库类型为“普通版”或“高可用版”。

**操作步骤**

1. 访问 [UDB PostgreSQL 实例列表](https://console.ucloud.cn/postgresql/postgresql)，找到目标实例，点击“操作”列中的“回档至新实例”按钮。

![](/images/1764042632014-e8a78a56-6063-49f6-9ee5-aa44894044b1.png)

2. 在回档弹窗中选择需要恢复的时间并确认，即可开始回档到新实例。云数据库 UDB PostgreSQL 支持数据库回档到新实例，但不支持指定库表。 

> 回档将会创建一个新的实例，该实例会按照正常计费规则收费，具体计费参考文档 [计费指南](/upgsql/price/bill) 。
